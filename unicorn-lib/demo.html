  • Accessibility: Focus automatically moves into dialog when opened and returns to trigger on close
  • Keyboard: Escape key closes dialog, Tab cycles through focusable elements
  • Mobile: On mobile, dialogs default to full width for better usability
  • Float variants: Great for sidebars, notifications, or slide-out panels
  • Unicorn mode: Adds magical rainbow gradient backdrop for special moments ✨
  • Custom actions: Use the footer slot to override default Cancel/Submit buttons
  • Scrollable content: Enable scrollable prop for long content with fixed dialog height

💡 Usage Guidelines

  • Filled variant: High-emphasis tags with white text on colored background - best for statuses
  • Outlined variant: Medium-emphasis tags with colored border and text - good for categories
  • Soft variant: Subtle tags with light tinted background - ideal for labels and metadata
  • Soft-outlined variant: Combines soft background with border - balanced emphasis with definition
  • Rainbow border: Gradient border wrapper for eye-catching tags - use rainbowColors prop for custom gradients
  • Glass (frosted): Translucent glassmorphism effect with backdrop blur - works best on colorful backgrounds
  • Glass (shiny): Enhanced glass with gradient shimmer and hover shine effect - premium look
  • Default icons: Success, Error, Warning, and Info colors automatically show semantic icons (filled, outlined, and soft-outlined)
  • Custom icons: Use prepend-icon and append-icon slots or props for custom icons
  • Colors: All colors provide proper contrast - filled tags use white text in light mode
  • Dismissible: Add dismiss functionality for removable filters or selections
  • Accessibility: WCAG 2.1 AA compliant with proper ARIA labels and keyboard support
  • Dark theme: All variants adapt automatically with proper contrast ratios
  • Mobile: Touch targets meet WCAG minimum 44x44px on mobile devices

💡 Usage Guidelines

  • Types: Snackbar (bottom/corner, 560px max, 2-line truncation), Banner (full-width top, 2-line truncation), In-context (inline, no truncation)
  • Variants: Success (green), Error (red), Warning (orange), Info (teal) - with automatic semantic icons
  • Backgrounds: Solid opaque colors using -light/-dark design tokens (no transparency)
  • Positioning: 13+ placement options (top-right, bottom-center, etc.) plus custom anchor-based positioning
  • Auto-dismiss: Optional timeout prop for automatic dismissal after specified milliseconds
  • Read more links: Add readMoreLink prop to provide additional information or actions
  • Dismissible: Control via dismissible prop - critical notifications should not be dismissible
  • Accessibility: WCAG 2.1 AA compliant with role="alert", aria-live (assertive for errors, polite for others), proper focus management
  • Screen readers: Error notifications use aria-live="assertive" for immediate announcement
  • Keyboard navigation: Dismiss button accessible via Tab, activated with Enter/Space, Escape also dismisses
  • Touch targets: Dismiss button meets 44x44px minimum on mobile for WCAG compliance
  • Dark theme: All variants adapt with proper contrast - backgrounds use -dark tokens in dark mode
  • Mobile-first: Responsive design with mobile-specific spacing and touch-friendly interactions
  • Best practices: Use snackbar for temporary feedback, banner for system-wide messages, in-context for form validation

Container System

Responsive container that centers content with responsive max-widths and consistent padding. Perfect for wrapping page layouts.

Standard Container (Responsive Max-Width)

.container

Max-width changes at breakpoints: 100% (xs/sm), 900px (md), 1185px (lg), 1785px (xl+)

Always centered with 1rem padding • Resize window to see responsive behavior

Fluid Container (Full Width)

.container.fluid

Always 100% width at all breakpoints • Still maintains 1rem padding

Container with Height Utilities

.container.h-screen

Combine container with height utilities like h-screen, min-h-screen, h-full
